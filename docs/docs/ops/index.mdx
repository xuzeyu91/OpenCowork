# Ops

Ops 面向“应用已经跑起来以后”的问题：配置如何生效、Cron 如何调度、SQLite 数据放在哪里、什么时候需要检查本地状态。

<SignalGrid>
  <Signal title="config" value="Settings + local files">
    UI 配置和本地文件共同决定运行时行为，排查时先确认当前 workspace。
  </Signal>
  <Signal title="schedule" value="Cron runtime">
    定时任务由 main process 调度，执行结果写入本地数据库和任务记录。
  </Signal>
  <Signal title="state" value="SQLite + resources">
    数据库、agents、commands、prompts 都在本地目录，升级时不要误删。
  </Signal>
</SignalGrid>

## Operator checklist

<Runbook>
  <RunbookStep title="Check config">
    确认 provider、channel、MCP、agent、workspace 设置与当前任务匹配。
  </RunbookStep>
  <RunbookStep title="Check runtime">
    确认 Electron main process、IPC handlers、renderer stores 的状态没有分叉。
  </RunbookStep>
  <RunbookStep title="Check persistence">
    确认 `~/.open-cowork/data.db` 和本地资源目录存在，并且没有被外部同步工具锁定。
  </RunbookStep>
</Runbook>
