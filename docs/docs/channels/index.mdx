# Channels

OpenCowork 的消息平台插件运行在 Electron main process。每个平台插件都扩展 `base-plugin-service.ts`，负责登录、连接、收消息、发消息和生命周期管理。

<SignalGrid>
  <Signal title="workplace" value="Feishu / DingTalk / WeCom">
    适合团队协作、企业机器人、卡片消息、审批流和群内任务分发。
  </Signal>
  <Signal title="community" value="Discord / Telegram / WhatsApp">
    适合社区频道、移动端通知、私聊和跨平台协作场景。
  </Signal>
  <Signal title="local cn" value="QQ / Weixin">
    适合中文个人号或本地社群入口，配置时更依赖平台登录状态。
  </Signal>
</SignalGrid>

## Core contract

<Runbook>
  <RunbookStep title="Start">
    `onStart()` 建立连接、登录或订阅事件，并把平台侧状态同步给 OpenCowork。
  </RunbookStep>
  <RunbookStep title="Receive">
    将平台消息规范化为内部消息事件，再交给 agent loop 或 session runtime。
  </RunbookStep>
  <RunbookStep title="Reply">
    支持发送文本、图片、文件或平台特定卡片，并处理失败重试和权限错误。
  </RunbookStep>
  <RunbookStep title="Stop">
    `onStop()` 关闭连接、清理 listener、释放登录态相关资源。
  </RunbookStep>
</Runbook>

## Choose a channel

- 团队协作优先：从 [Feishu](/docs/channels/feishu)、[DingTalk](/docs/channels/dingtalk) 或 [WeCom](/docs/channels/wecom) 开始。
- 社区和移动端优先：从 [Telegram](/docs/channels/telegram)、[Discord](/docs/channels/discord) 或 [WhatsApp](/docs/channels/whatsapp) 开始。
- 中文个人号场景：查看 [QQ](/docs/channels/qq) 或 [Weixin personal](/docs/channels/weixin-personal)。
